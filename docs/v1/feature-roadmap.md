# V1 功能规划

## 一、修改登录密码

```
1. 用户已登录，JWT 验证通过
       ↓
2. 输入当前登录密码 + 新登录密码
       ↓
3. 服务器验证当前密码正确
       ↓
4. 新登录密码 → Argon2 哈希 → 更新 passwordHash
       ↓
5. 完成（不影响主密码和所有 encryptedSecret）
```

**特点**：主密码不变，所有已加密数据无需重新加密。

## 二、修改主密码

```
1. 用户已解锁金库（已输入主密码，Ke 在内存中）
       ↓
2. 输入当前主密码 + 新主密码
       ↓
3. 客户端校验：新主密码 ≠ 登录密码
       ↓
4. 生成新的随机 salt
       ↓
5. 新主密码 + 新 salt → Argon2id → 派生新 Ke
       ↓
6. 用旧 Ke 解密所有 encryptedSecret → 得到明文
       ↓
7. 用新 Ke 重新加密所有数据 → 更新 encryptedSecret
       ↓
8. 更新 DB: 新 salt + 所有新 encryptedSecret
```

**警告**：此操作需要重新加密所有数据，可能耗时较长，应显示进度。

**备选方案（简化）**：

- 如果用户主密码泄露，直接重新派生 Ke 并重新加密
- 不支持"只改主密码不改加密数据"的原子操作
